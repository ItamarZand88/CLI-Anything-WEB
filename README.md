# CLI-Anything-Web

**A Claude Code plugin that generates production-grade Python CLIs for any web application by capturing its live HTTP traffic.**

Point it at a URL. Get a fully working CLI on your PATH — with auth, REPL mode, `--json` output, and tests.

## What it does

```
You: /cli-anything-web https://suno.com

Claude: Opens browser → captures API traffic → analyzes endpoints →
        generates cli-web-suno with auth, 14 commands, REPL mode, tests

You: cli-web-suno songs generate --description "jazz ballad about rain"
```

The plugin works through an 8-phase pipeline:

1. **Capture** — Opens a browser with playwright-cli, assesses the site (framework, protections, API shape), then records all HTTP traffic while systematically exercising the app
2. **Methodology** — Analyzes captured traffic, maps the API surface, designs the CLI architecture
3. **Testing** — Generates unit and E2E tests
4. **Standards** — Validates against publication standards, runs smoke tests, produces a pip-installable package

## Quick start

### Prerequisites

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) with plugin support
- [Node.js](https://nodejs.org/) (for playwright-cli)
- Python 3.10+

### Install the plugin

Run these two commands inside Claude Code:

```
/plugin marketplace add ItamarZand88/CLI-Anything-WEB
/plugin install cli-anything-web
/reload-plugins
```

That's it. The plugin is now available in any Claude Code session.

### Generate a CLI

```bash
/cli-anything-web https://monday.com
```

The agent opens a browser, asks you to log in if needed, captures traffic, and generates a complete CLI.

## Generated CLI features

Every CLI the plugin generates includes:

- **Click commands** — `cli-web-<app> <group> <command> [options]`
- **REPL mode** — Run without args for an interactive shell with history and autocomplete
- **`--json` output** — Machine-readable output for agent pipelines
- **Auth management** — Browser login, cookie import, token refresh
- **Tests** — Unit tests (mocked) + E2E tests (live API)
- **`setup.py`** — `pip install -e .` puts it on your PATH

## Commands

| Command | Description |
|---------|-------------|
| `/cli-anything-web <url>` | Full pipeline — capture, analyze, generate CLI |
| `/cli-anything-web:record <url>` | Capture traffic only |
| `/cli-anything-web:refine <path>` | Expand coverage of existing CLI |
| `/cli-anything-web:test <path>` | Run tests and update TEST.md |
| `/cli-anything-web:validate <path>` | Validate against standards |
| `/cli-anything-web:list` | List generated CLIs |

## How it works under the hood

The plugin instructs Claude to:

1. Launch a browser with `playwright-cli` (handles Chrome coexistence, auth, SPAs)
2. Run a quick site assessment — detect framework (Next.js, Nuxt, etc.), check for protections (Cloudflare, CAPTCHAs), discover API patterns
3. Start trace recording, then systematically explore READ and WRITE operations
4. Parse the trace into a structured API surface document
5. Generate a Python CLI following strict architectural patterns (namespace packages, Click groups, httpx client, session management)
6. Generate tests, validate against standards, produce a pip-installable package
